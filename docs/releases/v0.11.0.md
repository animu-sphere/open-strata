# v0.11.0 — producer-side correctness + Linux runtime portability

## Objective

The first *produce*-side hardening pass: make the producer path boring before the
trust-policy ladder layers on top. Proved public Windows and Linux runtime
consumption by digest, and fixed one hard ABI-labeling blocker plus a cluster of
producer rough edges.

## Shipped capabilities

- **Real glibc floor for Linux runtimes (P0 blocker):** a streaming ELF scanner
  (`ost_build::glibc`) computes the real maximum `GLIBC_x.y` from packed binaries
  and stamps that floor onto the artifact `target`, recording the drift — closing a
  false `--require-target` pass on a fabricated label.
- **`ost artifact push` against GHCR (P0):** fixed the double-`sha256:` producer
  bug; denied writes get an auth-mode-keyed hint; docs mark `OST_REGISTRY_TOKEN`
  pull-only.
- **Linux symlink preservation in export (P1):** safe, relative, in-tree symlinks
  survive as link entries (soname chains stay links, not copies); absolute /
  `..`-escaping links stay a hard error, with a pre-extraction safety gate.
- **Quieter/actionable package-stage fallback (P1):** `STAGE_FALLBACK` names locked
  siblings and points at `--clean-stage` (`STAGE_CLEANED` accounting).
- **Declarative repo smoke tests in generated source CI (P2):** a validated
  `source_checks:` list renders as steps after the pyramid, before packaging.
- **Docs + preflight retest (P2):** Linux `--build` prerequisites documented;
  build-host dep preflight verified with a unit-tested `missing_dep_warning`.

## Known limitations

A real GHCR round-trip of the `ost artifact push` user/password path (needs live
credentials) remained open — tracked in [roadmap/current.md](../roadmap/current.md).
