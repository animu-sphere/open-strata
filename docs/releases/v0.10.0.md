# v0.10.0 — OCI publish foundation + runtime-completeness closure

## Objective

The first *produce*-side slice of remote transport. v0.9.0 made *pulling* an OCI
runtime first-class; v0.10.0 adds the producer verb and fixes three ways a
from-source runtime was not yet publishable.

## Shipped capabilities

- **Runtime completeness (P0):** `runtime pull --build` stamps the real OpenUSD
  version and drift recovery actually recovers a `build`-source runtime; a runtime
  that bundles schema tooling ships its Python deps (`jinja2` + `MarkupSafe`);
  `runtime export --slim` retains top-level `resources/`; an optional build-dep
  preflight.
- **`ost artifact push` (P1):** a write-capable OCI backend behind the existing
  `ArtifactTransport` contract emitting the exact OCI layout/media-types
  `ost artifact pull` expects; atomic / content-addressed / idempotent; prints the
  immutable OCI digest; every digest in `--json`; credential-store / env
  documentation.
- **`ost uv` shadow-dependency diagnosis:** warn/refuse app-local deps that shadow
  ABI-sensitive runtime packages.
