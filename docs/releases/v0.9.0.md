# v0.9.0 — remote artifact transport + hosted source-CI closure + macOS robustness

## Objective

Close the hosted-lane bootstrap gap (a hosted runner had nothing to install `ost`
with and nothing to seed the runtime SDK artifact) and harden macOS plugin builds.
Shipped and demonstrated on real CI: a migrated hosted `windows-2022` PR lane went
green end to end.

## Shipped capabilities

- **Transport abstraction + read-only OCI pull:** an `ArtifactTransport` contract
  with the filesystem store and a read-only OCI backend (GHCR-class);
  `ost artifact resolve | pull` with digest-pin enforcement, the full verification
  chain before an atomic local import, JSON pull evidence, and stable `ARTIFACT_*`
  codes.
- **CI contract + generated hosted bootstrap:** an `openstrata.ci.yaml`
  `runtime_remote` reference (OCI uri + expected digest); `ost ci generate github`
  renders a pinned, checksum-verified `ost` install step and the digest-pinned
  `ost artifact pull`, with `actions/cache` keyed by digest (speed, never
  correctness).
- **Runtime export ergonomics:** a slim/SDK-profile export, multithreaded zstd by
  default (`--jobs`, `--level`), and throttled progress.
- **macOS plugin-build robustness:** Python resolved from the runtime interpreter;
  schema-regeneration env scoped to the runtime's plugin registry; early failure on
  a foreign-platform `LibraryPath`; per-phase build-failure attribution.
