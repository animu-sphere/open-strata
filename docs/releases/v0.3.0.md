# v0.3.0 — Phase 4b plugin-harness dogfooding

## Objective

Make the file-format plugin bundle path solid end to end, from the first
downstream dogfooding round.

## Shipped capabilities

- Relative-path `ost plugin build | test`; MSVC bootstrap inside `ost plugin
  build`.
- Loadable `plugInfo.json` (target-configured library path/suffix); real USD
  version detection for adopted runtimes.
- `ost plugin package` artifacts.
- `fmt` / `clippy` / `test` CI gates.
- Plugin bundle `license` field.
