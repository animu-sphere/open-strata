# v0.6.0 — artifact registry + publishable plugin CI

## Objective

The first practical Phase 6 slice: make runtime/plugin/package artifacts
addressable by digest, publish plugin outputs into a local registry, and use those
artifacts as the source of truth for a runtime×plugin CI matrix.

## Shipped capabilities

- **Artifact store MVP:** local content-addressed store + registry index
  (`ost-artifact`); `tar.zst` + manifest + checksums + validation report as the
  canonical bundle; `ost artifact import|export|list|show|verify`; the
  `RuntimeSource::Artifact` path (`ost runtime export` / `ost runtime pull
  --from-artifact <digest>`).
- **Plugin publish MVP:** `ost plugin publish` (refuses missing
  validation/provenance/license/notices with stable codes; publishes by digest).
- **CI matrix MVP:** `ost ci init | validate [--resolve] | generate github` —
  explicit support lines (runtime digest × plugin digest × target/profile), never
  a naive Cartesian product.
- **Compiled co-located schema UX:** `ost plugin schema add`; adopted-runtime
  drift repair (`ost runtime repair`) with copy-paste fixes in every drift report.
