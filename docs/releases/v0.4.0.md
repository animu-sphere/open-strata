# v0.4.0 — the schema plugin kind

## Objective

Add `usd-schema` as a first-class plugin kind and close the remaining Phase 4
scaffold/diagnostic gaps.

## Shipped capabilities

- Codeless `usd-schema` template + codeless-aware L0 doctor (registers on a real
  runtime).
- Schema test contract (L2 registration / L4 apply-roundtrip), verified e2e on
  OpenUSD 26.08.
- Co-hosting a schema in an existing bundle; per-variant `cxx_abi`; the
  `usdGenSchema` regenerate build step; the `usdGenSchema` `Types` merge into a
  co-hosting bundle's existing `plugInfo.json`.
- Phase 4 close-out: P3 repo-shape scaffold (`ost init --template
  plugin-workspace`) and `ost doctor` structuring (structured issues + runtime
  `kind`).

## Known limitations

The compiled (non-codeless) schema variant — the typed-C++ importer API — was
deferred to a later release; the codeless + co-hosting paths cover the
data-contract use case.
