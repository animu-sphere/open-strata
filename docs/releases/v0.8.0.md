# v0.8.0 — packaging reruns survive transient locked stage trees

## Objective

A narrow follow-up from dogfooding report #9: make packaging robust to transient
Windows file locks so CI reruns do not fail on a stale stage tree.

## Shipped capabilities

- `ost plugin package` and `ost package` reset staging directories with bounded
  retry, clear read-only entries during cleanup, and fall back to a fresh sibling
  stage when scanners/indexers still hold old files open.
- The fallback is visible as a structured `STAGE_FALLBACK` warning, so CI keeps
  moving without hiding the host condition.
