# v0.18.0 - evidence integrity and ecosystem documentation

## Objective

Two v0.17.0 dogfooding passes — hdMerlin and usd-vrm-plugins — surfaced the same
defect class at two layers: a PASS, or a success report, that is not bound to a
completed, owning producer. v0.17.0 made a *file* tell the truth about a build.
v0.18.0 extends that to whole operations: evidence import, generated CI gates,
renderer producer sessions, target ownership, test runs, and the dependency
closure a packaged workspace ships.

The release is corrective. No new DCC or host surface ships here; the previously
scheduled DCC host milestone moved to v0.20.0, behind the v0.19.0 Formation
composition milestone.

## Completed capabilities

- **Artifact evidence attaches and gates honestly.** `ArtifactStore::import`
  attaches sidecar SBOM/provenance that verifies against the stored archive
  digest, whatever the producer manifest says — the equality guard previously
  stranded evidence on any machine that already held the digest, with no
  repo-side cause. Caller-supplied evidence is never dropped silently: refusals
  are coded errors with non-zero exits, and import outcomes surface
  `evidence_attached` / `evidence_skipped`. `require_evidence` is declarable per
  cell and document-wide in `openstrata.ci.yaml`, `ost ci generate` warns when a
  pin cannot satisfy the gate it just rendered, `ost ci validate` fails fast on
  the same condition, and `ost artifact rm <digest>` is the supported registry
  reset for a machine holding a pre-evidence digest.
- **One completed producer behind every renderer PASS.** Renderer report
  overlays record a producer session — id, kind, target, start and completion
  times, final state — and a successful overlay is published by atomic rename
  only after its containing command completes. `ost renderer merge` refuses PASS
  assertions from failed, incomplete, mismatched, or superseded sessions, and
  `ost validate` names the producer behind each assertion.
- **Truthful artifact records.** `record.producer` is the tool that *built* an
  artifact, read from the dist manifest, and `imported_by` is what the field
  previously held: whoever created the registry entry. The same image no longer
  reads `ost 0.10.0` on one machine and `ost 0.17.0` on another.
- **OS-backed exclusive target leases.** Configure, build, output verification
  and completion publication hold one lease for the whole sequence. The
  exclusion is a kernel file lock — `flock` on Unix, an exclusive share mode on
  Windows — not a PID-file heuristic, so a builder killed by CI or a lost power
  rail releases it instead of wedging the target. A second writer's behavior is
  chosen, never implicit: `--on-busy fail` (default) stops with `TARGET_BUSY`
  naming the holder, `wait` queues under `--busy-timeout`, `read-only` proceeds
  without taking the lease. Stale-owner recovery verifies process identity
  before takeover and distinguishes an exited owner from a recycled PID and from
  a record written on another host. Completion records name their owning
  invocation.
- **Managed test lifecycle.** `ost test` runs a target's tests under the runtime
  that built it, propagating the runtime and MSVC environment, and reading the
  configuration and generator from the build completion record rather than
  accepting them again. Timeouts apply per test and over the whole run, and the
  overall one tears down the process tree. Results are counted from CTest's
  JUnit report, not scraped from its console output. An empty suite is a
  failure, not a quiet success: `tested` is its own claim in `ost validate`,
  distinct from `built`, `packaged` and host-side checks, and it is bound to the
  build it exercised by a fingerprint over that build's identity.
- **External build provenance.** `ost external import` inspects a build tree's
  own `CMakeCache.txt` and records source root, build directory, runtime
  id/digest/root, OpenUSD version, compiler, CRT, Python ABI, generator and
  configuration, digested against the exact cache entries it came from. On a
  full identity match `ost validate --build-dir` reports `runtime-compatible`;
  `configured` and `built` stay skipped, because they claim OpenStrata did the
  work.
- **Workspace packaging closure.** `ost plugin package --workspace` packages
  every discovered bundle in dependency order against the same validated graph
  `plugin test --workspace` checks. `dependencies.json` records resolved
  `bundles` the way it already recorded `libraries`, and the closure ships
  inside the artifact, so a consumer detects a missing provider from the
  manifest instead of at schema-application time. `--from-package` composes with
  `--workspace`, testing each bundle against the *extracted* trees of its
  dependencies. Staged manifest and `dependencies.json` paths are
  forward-slashed.
- **Provenance, pin, camera and advice ergonomics.** `--json` failures mirror
  their code and message to stderr, so a redirected stdout cannot hide them.
  `ost runtime export --build-metadata <file>` lets a non-GitHub-Actions
  producer emit provenance under the same required-fields rule. `artifact push`
  states which of its two digests churns on republish. `renderer view` selects a
  camera from the scene instead of asserting `/Camera`. `ost doctor` scopes its
  next actions to the profile and any `--capability` named, and explains what a
  real runtime would change — including when the answer is "nothing you
  selected". `runtime export --slim` warns when it drops a tree the shipped
  CMake config still references.
- **Reference projects and Formation design documentation.**
  `animu-sphere/usd-vrm-plugins` and `animu-sphere/hydra-merlin` are documented
  as reference projects, with the cross-repository Formation model specified as
  a v0.19.0 design target. Documentation only: no `ost formation` surface ships
  here.

## Compatibility notes

- Renderer reports written before v0.18.0 carry no producer session and
  therefore cannot contribute a PASS. That refusal is the fix, not a regression;
  re-run the producing check to obtain an owned report.
- `ost build` and `ost configure` take an exclusive target lease and fail with
  `TARGET_BUSY` (exit 4) when another invocation holds it. Concurrent
  invocations against one target that previously interleaved will now stop;
  choose `--on-busy wait` or `--on-busy read-only` deliberately. Managed builds
  such as `renderer view` default to `wait`.
- Artifact records written before v0.18.0 deserialize with no `imported_by`,
  which identifies them unambiguously; their legacy `producer` value migrates to
  the field it was always describing and origin returns to unknown. `artifact
  show` says "unrecorded" rather than naming a producer it cannot vouch for.
- `ost test` refuses a target with no registered tests rather than publishing a
  `tested` record asserting "0 of 0 passed". Projects using `ctest` for an
  intentionally empty suite should not call `ost test` on it.
- `renderer view --camera` no longer defaults to `/Camera`. Scripts that relied
  on the implicit value should pass it explicitly; it is honored when the scene
  contains it and reported as unhonored when it does not.
- `--json` failures now also write one line to stderr. Consumers that asserted
  an empty stderr on failure should assert on stdout, which remains exactly one
  parseable document.
- The generated evidence gate defaults to `require_evidence: all`. Upgrading
  never weakens a gate; a repository mid-migration declares what its pins can
  actually meet instead of being locked out.

## Known limitations / remaining follow-ups

- The aggregate workspace product artifact is deliberately deferred. Defining
  one means deciding how a consumer installs and pins a *set* rather than a
  bundle, which belongs with the v0.19.0 Formation milestone. v0.18.0 ships
  per-bundle artifacts packaged as a closure.
- `ost renderer view` opt-in detached mode — a durable session recording process
  identity, readiness and exit state, with host warnings classified separately
  from plugin-discovery, renderer-selection, scene-open and first-frame failures
  — remains outstanding. Automatic camera selection and the reported selection
  reason ship here.
- External build provenance reads a readable CMake cache. A tree whose cache is
  absent cannot be imported, and camera inspection of binary `.usdc`/`.usdz`
  layers reports "could not inspect" rather than guessing.
- Evidence for plugin artifacts published before evidence existed still has no
  republish story equivalent to the runtime one; the plugin-package
  republish/attach path is now unblocked by the import fix and remains to be
  defined.
- Carry-over acceptance from v0.16.0/v0.17.0 remains: the public macOS runtime
  republish, the GHCR push round trip, the hosted `vrmSchema` L5 matrix, and the
  first downstream OIDC + live GHCR release-lane round trip.
- Renderer manifests and reports remain `v1alpha1`; strict unknown-field
  rejection is intentional while the contract is applied to more projects.

## Evidence

- Rust unit and CLI integration tests cover: cross-process lease acquisition,
  busy refusal naming the holder, wait timeout, read-only attach and stale-owner
  takeover; test-completion fingerprint binding and empty-suite refusal;
  external provenance import, foreign-runtime refusal and cache-digest
  invalidation on reconfigure; whole-workspace topological packaging order and
  the recorded bundle closure; forward-slashed dependency paths; build-metadata
  field validation; camera selection across present, missing, and
  un-inspectable scenes; profile-scoped doctor advice; and slim-export layout
  reference detection.
- The lease, `ost test`, `ost external import` and `ost validate` paths were
  additionally exercised end-to-end against a real project, a real MSVC/Ninja
  toolchain and a real pulled OpenUSD 26.05 runtime on Windows, including a
  genuine reconfigure to confirm imported provenance stops verifying.
- Local release gate on Windows passed: `cargo check --workspace`,
  `cargo fmt --all -- --check`,
  `cargo clippy --workspace --all-targets --locked -- -D warnings`,
  `cargo test --workspace --locked`, `python scripts/check_doc_links.py .`, and
  `python scripts/check_docs_consistency.py .`.
