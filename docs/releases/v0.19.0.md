# v0.19.0 - reachable packages and Formation composition

## Objective

v0.19.0 closes the reach gap identified by v0.18.0 dogfooding and makes the
reference-project ecosystem composable. Packaged workspaces now carry and
expose the bytes they claim, while a strict Formation manifest can resolve,
pin, inspect, and run independently released components.

## Completed capabilities

- **Closed and reachable workspace packages.** Plugin packages stage transitive
  registration trees and libraries, publish portable activation metadata, and
  remain usable outside an OpenStrata source checkout.
- **Package-origin verification content.** Packaged tests carry versioned golden
  oracles and verify outputs by digest, including fail-closed behavior for
  missing or mismatched content.
- **Managed build and external provenance.** Package output is bound to the
  managed build completion record, and external-build records can resolve the
  real trees and producer sessions that exist on disk.
- **Aggregate product artifacts.** `ost plugin package --workspace --product`
  emits one digest-addressed product containing the exact member packages and a
  portable product manifest.
- **Named build intents.** Projects can declare typed build intents in
  `openstrata.toml`; build and renderer commands consume them with strict,
  deterministic validation.
- **Formation MVP.** The isolated `ost-formation` crate and CLI implement
  `ost formation resolve|inspect|lock|run`. Manifest and lock schemas are
  strict and versioned, component archives are digest-verified, compatibility
  and duplicate identities fail closed, and runs record exact component,
  runtime, lock, and environment evidence.
- **Contract consistency.** The release tightens unknown-key handling,
  packaged activation, stable diagnostics, redacted JSON output, and the
  renderer/session evidence paths used by downstream projects.

## Compatibility notes

- Formation `env` and `doctor` commands are not part of this MVP.
- Formation runs are foreground-only; detached session management is deferred.
- Formation component references are local/package-backed and digest-pinned;
  implicit downloads from untrusted sources are not introduced.
- DCC host discovery and adapters remain outside this release.

## Known limitations / remaining follow-ups

- The four first-party clean-machine, packaged-artifact Formation dogfoods are
  the remaining acceptance work for the composition model.
- DCC host integration moves to v0.20.0 and consumes Formation as its assembly
  layer.
- Broader hosted release-lane and cross-platform acceptance remains tracked in
  the roadmap and delivery history.

## Evidence

- Rust unit and CLI integration tests cover strict Formation parsing, portable
  locks, digest verification, product expansion, package-origin verification,
  build-intent validation, and the resolve → lock → inspect → run lifecycle.
- The release gate covers formatting, clippy, tests, licenses, documentation
  links, and documentation consistency.
