# v0.5.0 — schema authoring hardening + workspace ergonomics

## Objective

Make the v0.4 codeless and co-hosted schema paths reliable across
Windows/macOS and remove "works only if you know the trick" UX.

## Shipped capabilities

- UTF-8-forced schema generation; the `schema.library_prefix` double-prefix hint;
  per-target metadata adoption nudges.
- Runtime-version drift reporting across `show` / `validate` / doctor (human +
  `--json`).
- The discoverable `usd-plugin-workspace` template alias; `plugins/<name>/`
  workspace discovery; `ost plugin new` workspace guidance.
- macOS `runtime pull --build` notes for known source-build edges; build-hook
  groundwork for a compiled co-located flow.

## Known limitations

`plugin publish`, the runtime×plugin CI matrix, and content attribution remained
tied to the Phase 6 artifact/content store; a first-class compiled co-located
schema UX was a v0.6.0 follow-up.
