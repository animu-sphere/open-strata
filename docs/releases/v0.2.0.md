# v0.2.0 — machine-readable output + build hardening + security baseline

## Objective

Make `ost` CI-drivable and harden the build lifecycle and the security baseline.

## Shipped capabilities

- **`--json` output + error/exit-code contract:** a versioned
  `{ok, schema, data, warnings}` envelope with stable `error.code`/`category` and
  category-based exit codes.
- **Build-lifecycle hardening:** per-target trees, runtime-env-consistent CMake,
  and a build progress stream.
- **Security P0/P1 baseline (SEC-001…004):** package staging rejects unsafe files;
  plugin manifest paths stay in the bundle; safe atomic writes (`O_EXCL`); CI
  supply-chain pinning (every Action pinned to a full commit SHA).
